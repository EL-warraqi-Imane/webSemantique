1.Introduction générale

  • Le Web actuel permet d’accéder à une grande quantité d’informations
  • Les données sont principalement compréhensibles par les humains
  • Les machines ont cependant du mal à comprendre le sens des informations
  • Le Web sémantique apporte une solution à ce problème
2

2. Définition :

Web classique

Relie des documents par des liens hypertextes sans compréhension machine du contenu.

Web sémantique

Extension du Web actuel où l’information est munie d'une signification bien définie, permettant aux ordinateurs et aux personnes de mieux coopérer.

  • L’idée principale est de structurer les données de manière intelligente afin que les ordinateurs puissent les interpréter, les relier et les exploiter automatiquement.
  • Contrairement au Web traditionnel, où les données sont souvent non structurées, le Web sémantique repose sur des standards définis par le W3C.

4. Pourquoi le Web sémantique ?

Les problèmes du Web actuel

Recherche par mots-clés

Manque de précision et ambiguïté (ex: "Jaguar" : animal ou voiture ?).

Données en silos

Information non structurée et difficile à croiser automatiquement.

Aveuglement machine

L'ordinateur affiche le contenu mais ne comprend pas sa signification.

5.Principes et concepts clés

Le principe fondamental

Le Web sémantique permet aux machines de comprendre et exploiter le sens des données en ajoutant des métadonnées et des relations sémantiques.

Structure d'un graphe de connaissances

Rôle des métadonnées

Ajoutent du sens aux données en décrivant leur structure, leur contenu et leur contexte.

Exemple : Dublin Core, Schema.org, RDF

Relations sémantiques

Créent des liens significatifs entre les entités, permettant l'inférence et le raisonnement.

Exemple : "estParentDe", "travaillePour", "localiséÀ"

Graphes de connaissances

Représentent les connaissances sous forme de réseaux interconnectés de nœuds et de relations.

Exemple : Google Knowledge Graph, Wikidata

Raisonnement machine

Permet aux systèmes de déduire de nouvelles informations à partir des données existantes.

Exemple : Inférence OWL, moteurs de règles

6. Technologies : La Pile Sémantique

Architecture en couches (W3C)

Couche d'Interrogation & Ontologie

  • OWL (Ontology Web Language) : Description poussée de la sémantique et des règles.
  • SPARQL : Langage pour exécuter des requêtes précises dans les données.

RDF Schema (RDFS)

Rôle : Décrire la signification de la donnée.

Il permet de créer des vocabulaires légers (ex: définir qu'un "Chien" est un "Animal").

RDF & URI

  • RDF (Resource Description Framework) : Standard d'échange pour décrire la donnée (Triplets).
  • URI (Identifiant) : La base fondamentale pour coder et identifier chaque ressource de manière unique sur le Web.

7. Applications Concrètes

Où le Web Sémantique est-il utilisé aujourd'hui ?

Moteurs de Recherche (Knowledge Graph)

  • Avant : Recherche de chaînes de caractères ("Paris").
  • Maintenant : Compréhension de l'entité (Paris = Capitale).
  • Résultat : Les "Rich Snippets" et encadrés d'informations à droite des résultats Google.

E-Commerce & SEO (Schema.org)

  • Utilisation de balises sémantiques pour décrire les produits (Prix, Stock, Avis).
  • Permet aux moteurs de comparer automatiquement les offres et d'afficher les étoiles dans les résultats.

Santé & Recherche Scientifique

  • Interopérabilité : Faire communiquer des bases de données d'hôpitaux différents.
  • Unification des vocabulaires médicaux pour l'aide au diagnostic (ex: SNOMED CT).
